1) Product Overview (Plain-English)

- Observed: The app is a self-care/lifestyle companion with daily routines, glow-up plans, advice, boredom/detox activities, camera/filter flow, and profile/streak behavior, implemented under artifacts/girlyvibes/app/(tabs)/.
- Observed: Main user flows are tab-driven: home, routines, glowup, advice, chat, boredom, camera in artifacts/girlyvibes/app/(tabs)/_layout.tsx.
- Observed: Advice has detail pages (artifacts/girlyvibes/app/advice-detail.tsx), and profile is a separate route (artifacts/girlyvibes/app/profile.tsx).
- Observed: Chat UI currently shows "Coming Soon" and is not feature-complete in artifacts/girlyvibes/app/(tabs)/chat.tsx.
- Inferred: Product intent is lifestyle engagement + lightweight AI/telemetry backend, with some backend capabilities ahead of UI integration.

7) End-to-End Feature Flows (Step-by-Step)

- Home -> Content navigation:
  1) User opens app and lands on tabs/home.
  2) CTA navigates to routines/glowup/advice/profile via router in app/(tabs)/index.tsx.

- Routine tracking:
  1) User marks steps in app/(tabs)/routines.tsx.
  2) State updates in contexts/AppContext.tsx.
  3) Streak/progress persisted with AsyncStorage.

- Advice detail/favorites:
  1) Category selection and favorite toggle in app/(tabs)/advice.tsx.
  2) Detailed article view in app/advice-detail.tsx.

- Camera and photo logging:
  1) User picks/captures media in app/(tabs)/camera.tsx.
  2) Local save via media library.
  3) Optional backend log upload through lib/apiLogger.ts to /api/log/photo.

- App-open telemetry:
  1) On app layout init, logAppOpen() runs.
  2) Daily guard via AsyncStorage key.
  3) Sends payload to /api/log/open.

- Chat (Observed/Inferred):
  1) UI currently placeholder only.
  2) Backend SSE chat exists.
  3) Wiring pending between mobile tab and /api/chat.

10) Environment Variables and Configuration Matrix

- EXPO_PUBLIC_DOMAIN
  - Used by: artifacts/girlyvibes/lib/apiLogger.ts
  - Required: Yes for backend logging calls
  - Example: api.example.com
  - Impact if missing: malformed URL and request failures

- PORT
  - Used by: artifacts/api-server/src/index.ts, mobile build/serve scripts
  - Required: Yes for server startup
  - Example: 3000 (API)
  - Impact: server fails or binds unexpectedly

- NODE_ENV, LOG_LEVEL
  - Used by: artifacts/api-server/src/lib/logger.ts
  - Required: Optional but recommended
  - Impact: logging mode/verbosity differences

- RESEND_API_KEY
  - Used by: artifacts/api-server/src/routes/log/index.ts
  - Required: Yes for email log delivery
  - Impact: log routes fail to send email

- AI_INTEGRATIONS_GEMINI_BASE_URL, AI_INTEGRATIONS_GEMINI_API_KEY
  - Used by: lib/integrations-gemini-ai/src/client.ts
  - Required: Yes for chat integration startup
  - Impact: API process throws on startup if missing

- DATABASE_URL
  - Used by: lib/db/src/index.ts and lib/db/drizzle.config.ts
  - Required: Yes if DB features/migrations used
  - Impact: DB init/migration fails

- Replit variables (REPLIT_*)
  - Used by: mobile dev script/build pipeline
  - Required: context-dependent
  - Impact: local dev mismatch unless overridden

13) Security, Privacy, and Data Handling Review

- Observed: /log/photo accepts base64 image and forwards by email (artifacts/api-server/src/routes/log/index.ts).
- Observed: CORS currently permissive default in app.ts.
- Observed: chat rate limiting is memory-local and non-distributed.
- Observed: sensitive headers are redacted in logger config (good).
- Inferred risks: no clear auth/authorization for API routes, possible abuse of log/photo endpoints, privacy concerns with media email forwarding.
- Immediate hardening: add auth, size/type checks, strict CORS origins, durable rate limiting store, retention policy.

14) Known Issues / Blockers / Platform Pitfalls

- Observed: Windows shell incompatibilities:
  - root preinstall uses sh.
  - API dev script uses export.
  - mobile dev script relies on $VAR expansions.
- Observed: Expo runtime issue encountered: missing expo-blur caused mobile blue-screen crash.
- Observed: Expo interactive prompt ("Log in / Proceed anonymously") can block startup and make Expo Go fail if not answered.
- Observed: package version mismatch warnings around Expo dependencies can create instability.
- Inferred: repo is optimized for Replit/Linux-like flow; Windows requires command adjustments.

15) Practical Onboarding Guide (First 60 Minutes)

- 0-10 min (read):
  - Review root package.json, pnpm-workspace.yaml.
  - Inspect mobile routes in artifacts/girlyvibes/app/.
  - Inspect API routes in artifacts/api-server/src/routes/.

- 10-25 min (install + mobile run):
  - corepack pnpm install --ignore-scripts
  - cd artifacts/girlyvibes
  - corepack pnpm exec expo start --lan --clear
  - choose Proceed anonymously when prompted
  - scan QR via Expo Go (same Wi-Fi)

- 25-35 min (API run):
  - Set required env vars (PORT, Gemini keys, RESEND_API_KEY).
  - cd artifacts/api-server
  - Windows-safe start:
    - $env:NODE_ENV='development'
    - corepack pnpm run build
    - corepack pnpm run start

- 35-50 min (verify core flows):
  - Navigate tabs and test routines/advice/camera.
  - Confirm API logs for /api/log/open and /api/log/photo.
  - Validate health endpoint.

- 50-60 min (stabilize):
  - Align Expo package versions with SDK expectations.
  - Document .env template.
  - Note known gaps (chat UI wiring, OpenAPI drift).
